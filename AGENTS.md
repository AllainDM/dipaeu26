## 1. Обновление лога изменений

Лог предназначен для игроков — в него записываются только изменения **игровых данных**: новые ходы, карты, таблицы, правила, обновления списка игроков и т.п.

Технические изменения (стили, вёрстка, кнопки, рефакторинг кода) в лог **не вносятся**.

- Запись вставляется после `<p>История изменений:</p>`, перед первой существующей записью.
- Формат: `<p>DD.MM.YYYY: Описание изменения.</p>`
- Дата — текущая, формат `день.месяц.год` (например `24.06.2026`).
- Описание — краткое, **на русском языке**, в стиле существующих записей.
- Изменения одного дня могут быть как сгруппированы в одну строку, так и разнесены в отдельные записи с одной датой — следуй стилю уже существующих записей в логе.

**Примеры (что нужно писать):**
```html
<p>24.06.2026: Добавлены 14-24 ходы.</p>
<p>15.02.2026: Добавлены карты: политическая, климат, этносы, ресурсы. Добавлен список игроков с ссылками.</p>
```

**Примеры (что НЕ нужно писать):**
```html
<p>24.06.2026: Поправлены стили кнопок.</p>
<p>24.06.2026: Переработана вёрстка таблиц.</p>
```

## 3. Соглашения по коду

- **Язык**: контент, комментарии, коммиты — **русский**.
- **HTML**: `<!DOCTYPE html>`, `<meta charset="UTF-8">`, `<link rel="stylesheet" href="style.css">`, `<script src="analytics.js"></script>` в конце `<body>`.
- **Навигация**: `<header>` → `<ul class="mainmenu">` — копируется на все страницы.
- **Активные пункты меню**: `Правила`, `Ходы`, `Карты`, `Таблицы`, `Ссылки`, `Лог`.
- **Закомментированы** (не трогать без явной просьбы): `Игроки`, `Отзыв`.
- **CSS**: единый `style.css`. Локальные стили — через `<style>` в `<head>`.
- **JS**: vanilla, `const`/`let`, отступы 4 пробела.
- **Закомментированный код**: не удалять, не раскомментировать без явной просьбы.

## 5. Интерактивная таблица построек (`buildings.html`)

Данные берутся из `dev/buildings.xlsx`. После изменения файла нужно перегенерировать `buildings.js`:

```bash
python3 dev/xlsx2js.py
```

Скрипт читает 11 колонок xlsx (заголовки: Категория — Доход) и создаёт `buildings.js` с массивом `BUILDINGS` и всей интерактивной логикой (фильтры, сортировка, цвета).

**Зависимость:** `python3` + `openpyxl`.

---

## 6. Калькулятор БО (`dev/Kalkulyator_BO-2.xlsx`)

Файл `dev/Kalkulyator_BO-2.xlsx` — исходник для расчёта боёв (БО). Содержит:
- Юниты и их базовые характеристики
- Бонусы/штрафы terrain (тип местности) — матрицы ATK_MATRIX и DEF_MATRIX
- Бонусы/штрафы атака/оборона
- Бонусы перков и скилов командира
- Штрафы за атаку через реку (−10%) и десант (−30%)
- Расчёт итогового БО

**Веб-версия (источник истины для сайта):** `bo.js` + `bo.html`.  
Логика веб-версии должна соответствовать **тексту правил** (`rules/05army.html`), а не xlsx.

## 7. Генерация HTML-страниц правил (`rules/*.html`)

Исходник правил — `.typ`-файлы в отдельном репозитории (не в этом).  
После изменения `rules/*.typ` нужно перегенерировать `rules/*.html`:

```bash
python3 dev/typst2html.py /путь/к/папке/с/typst
```

Скрипт читает каждый `rules/*.typ` из репозитория typst, конвертирует разметку в HTML и раскладывает по страницам в `rules/`:

| Файл | Источник `.typ` | Раздел |
|------|----------------|--------|
| `contents.html` | `intro.typ` + оглавление | Введение + оглавление |
| `01concepts.html` | `common_things.typ` | 1. Общие понятия |
| `02states.html` | `country.typ` | 2. Государство |
| `03persons.html` | `characters.typ` | 3. Персонажи |
| `04trade.html` | `tradeship.typ` | 4. Торговля |
| `05army.html` | `military.typ` | 5. Военное дело |
| `06diplomacy.html` | `diplomacy.typ` | 6. Дипломатия |
| `07espionage.html` | `espionage.typ` | 7. Шпионаж |
| `08piracy.html` | `piracy.typ` | 8. Пиратство |
| `09events.html` | `events.typ` | 9. Случайные события |
| `10sri.html` | `sri.typ` | 10. СРИ |
| `11papa.html` | `papa.typ` | 11. Папа Римский |

**Поддерживаемые конструкции typst:**
- Заголовки `=`/`==`/`===` → `<h1>`/`<h2>`/`<h3>`
- Метки `<label>` → `id="label"` у заголовка
- `*bold*`, `_italic_`, списки `-`/`+`/`1.`
- `#important[body]` → `<div class="important">`
- `#example[body]` → `<div class="example">`
- `#emph[body]` → `<div class="emph">`
- `$math$` → `<code class="math">`
- `#table(columns: N, [cell1], [cell2], …)` → `<table>`
- `@label` → ссылка на `#label`
- `---` → длинное тире

**Зависимости:** только `python3` (стандартная библиотека).
